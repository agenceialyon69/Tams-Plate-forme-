# TAMS — Non-objectifs

> Ce que TAMS n'est pas et ne doit pas devenir.

## TAMS n'est pas

- **Pas un dashboard** — TAMS aide à décider et agir, pas juste afficher des données.
- **Pas un CRM** — Le CRM léger est un sous-module de Work OS, pas le produit.
- **Pas Notion** — Pas un éditeur de documents générique.
- **Pas un SaaS générique** — Pas de multi-tenant, pas de plans payants.
- **Pas un clone de ChatGPT** — Le chat est intégré au système, pas isolé.
- **Pas un admin dashboard** — Jamais.

## Ce que TAMS ne fait pas

- Pas de multi-utilisateur (single-user : Mohamed).
- Pas de plans payants (gratuit uniquement).
- Pas de feature theater (écrans décoratifs sans valeur).
- Pas de modules par habitude (chaque module doit justifier son existence).
- Pas de raccourcis (chaque étape doit être validée).
- Pas de faux DONE (preuve runtime obligatoire).

## Ce que TAMS évite

- La complexité inutile.
- Les doublons de fonctionnalité.
- Les dépendances payantes obligatoires.
- Les écrans qu'on n'ouvrirait pas tous les jours.
- L'effet "collection de modules séparés".
