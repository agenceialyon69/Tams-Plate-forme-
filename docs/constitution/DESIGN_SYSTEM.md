# TAMS — Design System

> Principes de design premium pour TAMS.

## Inspirations

Apple, Linear, Arc, Notion, Raycast, ChatGPT, Perplexity.

Mais jamais un admin dashboard. Jamais.

## Principes

- **Mobile-first** : le mobile doit être excellent, pas "acceptable".
- **Clarté** : chaque écran doit aider à décider, agir et comprendre vite.
- **Hiérarchie** : typographie + espacement pour guider l'œil.
- **Espace blanc** : réduire la charge cognitive.
- **Progressive disclosure** : révéler les actions secondaires contextuellement.
- **Single Responsibility** : un écran = un but.

## Stack UI

- Tailwind CSS 4 + shadcn/ui (Radix primitives)
- Framer Motion (animations)
- Lucide React (icônes)
- Sonner (toasts)
- Vaul (drawers mobile)
- Recharts (charts)

## Règles mobile

- Navigation basse (BottomNav)
- Safe areas
- Gestes
- Keyboard handling
- Responsive parfait
- 60 FPS si possible
- Lisibilité excellente
- Interactions naturelles

## Couleurs

Ne jamais utiliser violet/indigo sauf demande explicite. Privilégier neutres, bleus, verts.

## Règle de l'écran quotidien

Avant chaque écran, se demander : "Est-ce que je l'ouvrirais tous les jours ?"
Si non, fusionner, simplifier ou supprimer.
